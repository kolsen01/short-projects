Security Report

Introduction

The product is a web application that renders a location map using client and server side data. Unforutnatley, the web application is riddled with security and privacy vulnerabilities. I am hired to resolve the security and privacy issues in the applications. For reference, the following is the link to the web application we are testing: https://landmark-locator.herokuapp.com/ .

Methodology

The first step is to clean out any past documents in the collection (MongoDB database) for checkins. This allows for the following to be executed:

  • 1. Bypassing Restrictions on Input Choices 2. Testing Cross Site Scripting (XSS):

    Through injecting a script with curl, I will test if a response with be triggered from the web broswer. If it does, then there is a vulnerability in the web application.

    3. Privacy check

Abstract of findings

The following vulnerabilities were found: Cross Site Scripting (XSS), privacy errors for location and logins, and bypassing restrictions on input choices. For XSS, scripts were injected to login using curl. Specifically, adding an 'alert' popup and window location redirection was inserted to login via script. For privacy, there were two errors found. Changing the query string on the checkins.json page can display all logins of all users who have checked in and their location. Furthermore, the location of the user is not private. Testing input parameters, the web applicatoin resulted in an application error when not all of the parameters were satisfied.
